<objective>
Fix GovTrace web app crashes, 502 Bad Gateway errors, and slow first page loads by rewriting `apps/web/server.ts` from a buffered SSR wrapper (that crashes on any handler error) into a production-grade streaming SSR server with proper error boundaries, compile-time TypeScript, a dedicated /health endpoint, SSR data prefetch on the home page, Cache-Control for public pages, and gzip/brotli compression for text responses.

Purpose: The current server buffers the whole SSR response via `await response.text()` and has no try/catch around `handler()`, so any server-fn error kills the Node process — the root cause of 502s. Running `tsx server.ts` at runtime also parses TypeScript on every cold start. Combined with no SSR prefetching, the home page is slow because data fetches wait for client hydration.

Output: Six atomic commits, each independently `git revert`-able. Task 1 is the linchpin — if it breaks the build, halt and investigate before starting Task 2. Tasks 2-6 build on Task 1 but do not depend on each other for correctness, and each can be reverted on its own.
</objective>

   function pickEncoding(acceptEncoding: string | undefined): 'br' | 'gzip' | null {
     if (!acceptEncoding) return null
     const header = acceptEncoding.toLowerCase()
     if (header.includes('br')) return 'br'
     if (header.includes('gzip')) return 'gzip'
     return null
   }

   function isCompressibleType(contentType: string): boolean {
     if (COMPRESSIBLE_PREFIXES.some((prefix) => contentType.startsWith(prefix))) {

     const base = contentType.split(';')[0].trim()
