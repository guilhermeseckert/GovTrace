## What changed

### Task 1 — Bucket-aware top-N search (`apps/web/src/server-fns/search.ts`)

When the caller requests `type=all` AND the normalized query is substantive (>=2 chars), `searchEntities` now runs **three parallel bucketed queries** via `Promise.all`:

- **Politician bucket:** `entity_type = 'politician'`, `LIMIT 10`
- **Org-group bucket:** `entity_type IN ('organization', 'company', 'department')`, `LIMIT 10`
- **Person-group bucket:** `entity_type = 'person'`, `LIMIT 10`

Results are concatenated in that order, deduped by `id` via a `Set`, and capped at 30 before the downstream `getEntityCounts` call. Province / date filters are applied identically to each bucket. Single-bucket (`type !== 'all'`) and browse-mode (`query < 2`) branches are unchanged.

**Before / after for `trudeau` (`type=all`):**

- Before: pg_trgm similarity ranked all entities against the normalized query in one SELECT, so single-token shorter names (e.g. a donor named `JUSTIN`) could outscore the longer politician name `TRUDEAU, Justin` and bury it past page 1.
- After: the politician bucket's top 10 — which includes `TRUDEAU, Justin` — is placed first in the merged list, so politician matches surface within the first 30 results regardless of how many shorter donor/person hits exist.

### Task 2 — Friendly department names in aggregates (`apps/web/src/server-fns/entity-aggregates.ts`)

Imported `getDepartmentName` from `@/lib/department-codes` and wrapped both department assignments in `computeAggregates`:

- `primaryDepartment` — null-safe: raw code → friendly name, or `null` if no department
- `largestDeal.department` — null-safe: raw code → friendly name, or `null`

No SQL, cache-key, or TypeScript type-shape changes. DB still stores raw Open Canada codes as the source of truth; resolution happens at the TS boundary only.

**Before / after example:**

- Before: an entity with its largest contract from Public Services and Procurement Canada showed `primaryDepartment: "pwgsc-tpsgc"` on its aggregate card.
- After: same entity shows `primaryDepartment: "Public Services and Procurement Canada"`.

### Task 3 — Coolify deploy workaround docs (`docs/deploy-coolify.md`)

New 81-line markdown cheat sheet at the repo root. Five `##` sections:

1. **When Coolify auto-deploy no-ops** — explains the image-cache-reuse root cause
2. **Manual rebuild (SSH recipe)** — `docker compose build --no-cache web` + `up -d --no-deps --force-recreate web`
3. **Apply a Drizzle migration on prod** — `docker exec -i <pg-container> psql -U govtrace -d govtrace < packages/db/drizzle/XXXX_NAME.sql`
4. **Verify a fix is live** — gzip-aware curl recipes (`--compressed` or `| gunzip`)
5. **Signs Coolify deployed but didn't actually update code** — 4-item bullet checklist (image ID, container uptime, curl match, CACHED build layers)
