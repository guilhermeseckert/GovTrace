# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | neutral theme, standard radius (shadcn defaults — no custom preset string) |
| Component library | radix-ui (via shadcn) |

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline chip spacing |
| sm | 8px | Compact table cell padding, input inner spacing |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, tab content inset |
| xl | 32px | Layout column gaps, hero section vertical rhythm |
| 2xl | 48px | Major section breaks between profile tabs and header |
| 3xl | 64px | Landing page hero vertical center offset |

Exceptions:
- Touch targets for the "Flag an error" button and tab triggers: minimum 44px height (accessibility requirement, WCAG 2.5.5)
- Autocomplete dropdown item row height: 44px minimum for tap accuracy on mobile

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

Notes:
- Inter loaded via system font stack (`ui-sans-serif, system-ui, -apple-system, sans-serif`) — no external font request for performance
- Monospace (table numeric columns — amounts, counts, dates): `font-variant-numeric: tabular-nums` applied via Tailwind `tabular-nums` utility on all currency and count cells
- Data provenance timestamps (PROF-06): Label (14px, regular) + `text-muted-foreground` — no additional size exception

## Color

**Brand:** Deep government blue `#1a3a5c`

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` — light: `#ffffff` / dark: `#09090b` | Page backgrounds, main content area |
| Secondary (30%) | `hsl(var(--card))` — light: `#f9f9fb` / dark: `#18181b` | Cards, data table alternating rows, profile sidebar, autocomplete dropdown, tab panels |
| Accent (10%) | `#1a3a5c` (government blue, mapped to `--primary`) | Specific elements listed below only |
| Destructive | `hsl(var(--destructive))` — `#dc2626` | "Flag an error" confirm action only |

Accent (`#1a3a5c`) reserved for:
1. Primary CTA button: "Search the database"
2. Active tab indicator underline
3. Entity name link on search result cards (hover state)
4. Confidence badge border for High/green state
5. Autocomplete suggestion highlight on keyboard selection

Semantic status colors (not accent — standalone palette):
- AI confidence High: background `#16a34a` (green-600), text white — label "High confidence"
- AI confidence Medium: background `#d97706` (amber-600), text white — label "AI-verified"
- AI confidence Low: background `#dc2626` (red-600), text white — label "Unverified"

Dark mode: CSS variables approach (shadcn standard). All color values reference `hsl(var(--...))` tokens. No hardcoded hex in components except the brand blue mapped to `--primary` in `globals.css`.

## Pages and Key UI Surfaces

### 1. Landing Page

**Layout:** Single centered column, max-width 640px for hero, full-width stat bar below.

**Primary focal point:** The search bar — it is the first interactive element in the visual hierarchy and receives autofocus on page load.

**Hero section:**
- Tagline: "Follow the money. Connect the dots. Hold power accountable." — Display (28px, semibold, line-height 1.15), centered
- Search bar: full-width input with shadcn `Input` at height 52px, placeholder "Search politicians, companies, or people…", right-slotted "Search the database" button (primary, government blue)
- Search bar shadow: `shadow-md` on focus to indicate active state
- Below tagline: platform stats row — 4 stat chips showing record counts (e.g., "1.2M donations tracked") in Label (14px, regular)

**Stat chips:** 4 items in a horizontal flex row, centered, wraps at <640px to 2×2 grid. Each chip: number in semibold 20px + descriptor in regular 14px stacked vertically, separated by 24px gap.

### 2. Search Results Page

**Layout:** Two-column at ≥768px (filters sidebar 240px fixed left, results main right). Single column at <768px (filters collapse to a top filter bar).

**Results grouped by entity type** (SRCH-03): section headings at Heading (20px, semibold) with count badge. Order: Politicians → Companies/Organizations → People.

**Result card:**
- Entity name: Heading weight (semibold 16px), accent color on hover, links to profile
- Entity type badge: shadcn `Badge` variant outline, label size (14px)
- Summary counts row: 4 inline chips — "💰 12 donations", "📄 3 contracts", "🤝 2 lobbying", "💵 1 grant" — Label (14px, regular), muted foreground color
- AI confidence badge: right-aligned, 3-state (High/Medium/Low) — see color spec above
- Card padding: 16px, gap between cards: 8px

**Filter sidebar:**
- Entity type checkboxes (Politicians, Companies, People)
- Date range: two date inputs (From / To)
- Province select: shadcn `Select` with all 13 provinces + territories
- "Apply filters" button: secondary variant
- Filter count badge on mobile trigger button when filters active

**Empty results state:**
- Heading: "No matching entities found"
- Body: "Try a different spelling or a shorter name. Entity names may vary across government datasets."
- Subtext: "Searching for a company? Try searching without 'Inc.' or 'Ltd.'"

### 3. Entity Profile Page

**Layout:** Full-width header band (government blue background), content below in max-width 1200px centered container.

**Header band:**
- Entity name: Display (28px, semibold, white text on blue)
- Entity type label: Label (14px, regular, white/80% opacity)
- AI match confidence badge: inline-right of name on desktop, below name on mobile
- "Flag an error" button: ghost variant, white text, `Flag` lucide icon left, right-aligned in header

**AI Summary block** (PROF-02 — must be first visible content below header):
- Container: card with left border 4px solid accent blue, background secondary
- Summary text: Body (16px, regular, 1.5 line-height)
- Format — plain language with emoji icons per PROJECT.md:
  - `💰` Political donations
  - `📄` Government contracts
  - `🤝` Lobbying activity
  - `💵` Grants received
  - `🔗` Connections to other entities
- Disclaimer (AI-02, DSGN-06): italic Label (14px) below summary text — "Connections shown do not imply wrongdoing."
- "How do we write this summary?" link (AI-05): Label (14px), muted, opens modal with explanation

**Tabbed data sections** (PROF-03):
- Tab bar: shadcn `Tabs` component, 5 tabs: Donations | Contracts | Grants | Lobbying | Connections
- Tab label includes count badge: "Donations (142)"
- Active tab: accent blue underline 2px, semibold label weight
- Tab panel padding: 24px top

**Data tables** (PROF-04):
- shadcn `Table` component
- Sortable column headers: `ArrowUpDown` lucide icon right of label, clickable
- Pagination: shadcn `Pagination` — rows per page select (10/25/50), page controls
- Amount columns: `tabular-nums`, right-aligned
- Date columns: `tabular-nums`, format `YYYY-MM-DD`
- Source link column: `ExternalLink` lucide icon, 14px, links to government source (PROF-05)
- Mobile (<768px): tables collapse to stacked cards — each row becomes a card with label/value pairs

**Connections tab:**
- Relationship disclaimer banner (DSGN-06): amber background, `Info` icon, text "Connections shown do not imply wrongdoing." — displayed at top of tab panel, above the connections table
- Each connection row: connected entity name (link) + relationship type badge + total value + transaction count + date range

**Data provenance footer** (PROF-06):
- Below tabs: row of 5 dataset last-updated timestamps — "Elections Canada: last updated 2026-03-28" — Label (14px, regular, `text-muted-foreground`)

### 4. AI Match Confidence Badge

Three states (AI-04):

| State | Background | Text | Label | Icon |
|-------|-----------|------|-------|------|
| High | `#16a34a` | white | "High confidence" | `ShieldCheck` lucide |
| Medium | `#d97706` | white | "AI-verified" | `Shield` lucide |
| Low | `#dc2626` | white | "Unverified" | `ShieldAlert` lucide |

- Badge size: Label (14px), padding 4px 8px, border-radius 4px (rounded-sm)
- Expandable: clicking badge opens shadcn `Popover` showing confidence score (e.g., "0.82"), match method (e.g., "pg_trgm + Claude haiku-3.5"), and AI reasoning text
- "How is this calculated?" link in popover — muted, 14px

### 5. Flag an Error Modal (COMM-01, COMM-02)

- Trigger: "Flag an error" button (ghost, white, `Flag` icon) in entity profile header
- Opens: shadcn `Dialog` modal
- Title: "Flag an error" (Heading 20px)
- Body copy: "Help us improve GovTrace. If you believe this entity match is incorrect, let us know."
- Form fields:
  - Describe the error: `Textarea` (required), placeholder "e.g., This profile merges two different people with similar names"
  - Your email (optional): `Input`, placeholder "your@email.com — for follow-up only"
- Submit button: "Submit flag" (primary)
- Cancel: "Never mind" (ghost)
- Confirmation state (after submit): body replaced with "Thanks — your flag has been recorded. We review all submissions." + Close button
- Destructive confirmation: not required (flagging is additive, not destructive)

## Accessibility Requirements

- All form inputs have visible `<label>` elements (not placeholder-only)
- Color is never the sole differentiator — confidence badge states also differ by icon and label text (WCAG 1.4.1)
- Focus rings: shadcn default focus-visible ring (`ring-2 ring-offset-2 ring-primary`)
- Autocomplete dropdown: `role="listbox"`, items `role="option"`, `aria-selected` on active item
- Data tables: `<thead>` with `scope="col"`, sort direction communicated via `aria-sort`
- Disclaimer banner: `role="status"` with `aria-live="polite"` if dynamically inserted
- Skip-to-content link at top of every page targeting `#main-content`
- Keyboard: Tab/Shift+Tab navigates all interactive elements; Enter/Space activates buttons; Escape closes modals and dropdowns

## i18n Structure (DSGN-04)

All visible copy strings must be defined in a translation key file (`apps/web/src/i18n/en.ts`) rather than hardcoded in components. English-only for v1. Key prefix conventions:

- `search.*` — Landing and search results
- `profile.*` — Entity profile page
- `badge.*` — Confidence badge states
- `flag.*` — Flag an error flow
- `table.*` — Generic table copy (empty states, pagination)
- `common.*` — Shared (error states, loading)
