# Phase 4.2: Entity Profile Storytelling - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Transform the entity profile from a data-analyst view into a story-first experience. Normal people see plain English stories and connection cards. Power users toggle into raw data tables. No new data sources — reshapes how existing data is presented.

</domain>

<decisions>
## Implementation Decisions

### AI Summary Upgrade
- **D-01:** Rewrite `buildSummaryPrompt` in `summary.ts` to query `entity_connections` for actual named connections, amounts, and types — feed these to Claude so it writes cross-referenced stories ("Jane Fulton donated $500 to Mark Carney. Jane Fulton's employer received a $2.1M contract from Public Works.")
- **D-02:** Query top 20 connections by totalValue from `entity_connections` + join to `entities` for canonical names — pass as structured context to Claude prompt
- **D-03:** Cache the enriched summary in the existing `aiSummaries` table (same schema, richer content)
- **D-04:** Claude prompt must include: "Write in plain language a 9-year-old could follow. Use rounded numbers. Name specific people and companies. Connect dots across datasets."

### Connection Cards
- **D-05:** New `ConnectionCards` component replaces `ConnectionsTable` as the primary connections view
- **D-06:** Each card renders as a plain English sentence: "{Entity A} **{friendly verb}** {Entity B} — {amount}" using the existing `CONNECTION_LABELS` map from `ConnectionsTable.tsx` (line 59-65)
- **D-07:** Cards are color-coded by connection type using the existing color classes in `CONNECTION_LABELS`
- **D-08:** Cards are sorted by totalValue descending (biggest connections first)
- **D-09:** Show max 10 cards by default with "Show all N connections" expand

### Pattern Callouts
- **D-10:** Rule-based pattern detection via SQL GROUP BY on `entity_connections` — NOT AI-generated (free, fast, deterministic)
- **D-11:** New server function `getPatternCallouts` detects specific patterns:
  - Multiple donors to the same politician sharing an employer/organization
  - Entity receiving contracts from multiple departments
  - Temporal clustering (lobbying registration close to contract award)
  - High-value single connections (top 1% by value)
- **D-12:** Displayed as "Did you know?" cards with a one-liner explanation and link to the source records
- **D-13:** Max 3 callouts per entity (most interesting patterns only)
- **D-14:** Each callout includes a "Why this matters" one-liner

### Page Layout Restructure
- **D-15:** New default layout order: EntityHeader → AI Story → Pattern Callouts → Connection Cards → Network Graph → "Show detailed records" toggle
- **D-16:** Network Graph is the default visualization shown (most visual impact for relationship discovery)
- **D-17:** "Show detailed records" toggle reveals the full ProfileTabs with all 6 tabs (Donations, Contracts, Grants, Lobbying, Connections table, Visualizations)
- **D-18:** Toggle uses a clear label like "Show detailed records" with a ChevronDown icon
- **D-19:** Default view (story mode) must be readable on mobile without horizontal scrolling
- **D-20:** Toggle state does NOT persist — always starts in story mode (the grandpa view)

### Claude's Discretion
- Connection card visual design details (padding, border radius, shadow)
- Pattern callout card styling
- Exact Claude prompt wording beyond the constraints in D-04
- Number of connections to feed to Claude prompt (D-02 says 20, can adjust based on token limits)
- Animation/transition on toggle expand

</decisions>
