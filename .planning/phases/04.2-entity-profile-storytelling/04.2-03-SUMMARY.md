# Phase 04.2 Plan 03: Story-First Entity Profile Layout Summary

Story-first entity profile with AISummary, PatternCallouts, ConnectionCards, and inline NetworkGraph as the default view; raw data tables hidden behind a "Show detailed records" toggle (aria-expanded, rotating ChevronDown).

## What Was Built

The entity profile page (`apps/web/src/routes/entity/$id.tsx`) was restructured to a story-first layout per STORY-04 and STORY-05. The transformation:

**Default (story mode) view order:**
1. EntityHeader (existing)
2. AISummary enriched with named connections and amounts (Plan 01)
3. PatternCallouts "Did you know?" cards (Plan 02)
4. ConnectionCards plain English sentences with color-coded badges (Plan 02)
5. NetworkGraph inline (promoted from Visualizations sub-tab)
6. "Show detailed records" toggle button

**Behind toggle (power user mode):**
7. ProfileTabs with all 6 tabs (conditionally rendered — not CSS hidden)
8. Provenance (always visible)

Key implementation details:
- `useState(false)` — toggle always starts collapsed, no URL persistence
- `aria-expanded={showDetailedRecords}` on toggle button for accessibility
- `ChevronDown` with `rotate-180` class on open state
- i18n keys `showDetailedRecords` / `hideDetailedRecords` in `en.profile`
- Pending skeleton updated: summary skeleton + 2 card skeletons + graph skeleton area

**Mobile (STORY-05):**
- Container uses existing `max-w-[1200px] px-4` — no horizontal overflow
- ConnectionCards and PatternCallouts are vertical card stacks (Plan 02 design)
- NetworkGraph uses SVG viewBox for responsive containment
