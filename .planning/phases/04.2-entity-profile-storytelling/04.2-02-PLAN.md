**Part A: `apps/web/src/server-fns/patterns.ts`** — per D-10 through D-14:

1. Define the return type:
   ```typescript
   type PatternCallout = {
     id: string
     question: string      // "Did you know...?" phrasing
     whyItMatters: string   // one-liner explanation
     sourceEntityId: string // for linking to connected entity profile (MVP simplification per RESEARCH.md Open Question 3)
   }
   ```

2. Create `getPatternCallouts` as a `createServerFn({ method: 'GET' })` with input `z.object({ entityId: z.string().uuid() })`, returning `PatternCallout[]`.

3. Implement three pattern detection queries (D-11) using `Promise.all`:

   **Pattern A: Entity receiving contracts from multiple departments**
   - Query entityConnections where (entityAId = entityId OR entityBId = entityId) AND connectionType = 'vendor_to_department', group by the "other" entity side, count distinct connected entities
   - If count > 1: question = "Did you know? {entityName} has contracts with {count} different government departments", whyItMatters = "Multiple department relationships may indicate a large government vendor"
   - sourceEntityId = entityId (links back to self for the contracts tab)

   **Pattern B: High-value single connection (top 1% proxy: > $500K threshold)**
   - Query entityConnections where (entityAId = entityId OR entityBId = entityId) AND totalValue > 500000, order by totalValue desc, limit 1
   - If found: question = "Did you know? This entity has a ${formatAmount(value)} connection with {connectedEntityName}", whyItMatters = "This is among the highest-value connections in the dataset"
   - sourceEntityId = the connected entity ID

   **Pattern C: Temporal clustering (lobbying near contract, 90-day window)**
   - Use `db.execute(sql\`...\`)` for this since it requires a self-join across connection types
   - Find entity_connections rows where connectionType = 'lobbyist_to_official' or 'lobbyist_client_to_official' AND first_seen is within 90 days of a 'vendor_to_department' connection's first_seen for the same entity
   - Define `const TEMPORAL_CLUSTER_DAYS = 90` as a named constant
   - If found: question = "Did you know? Lobbying activity was registered within 90 days of a contract award", whyItMatters = "Temporal proximity between lobbying and contracts is a pattern civic researchers watch"
   - sourceEntityId = the connected entity from the vendor connection

   **Employer-sharing pattern (D-11 pattern 1):** Per RESEARCH.md Open Question 2, this requires JSONB extraction from raw donations table which is not indexed. SKIP this pattern for Phase 4.2 — use Pattern D instead:

   **Pattern D: Donor to multiple politicians**
   - Query entityConnections where (entityAId = entityId OR entityBId = entityId) AND connectionType = 'donor_to_party', count distinct connected entities
   - If count > 2: question = "Did you know? {entityName} has donated to {count} different political parties or politicians", whyItMatters = "Donations to multiple parties may reflect broad political engagement"

4. Collect non-null results from all 4 patterns, sort by value (monetary patterns first), return first 3 (D-13).

5. If no patterns found, return empty array.

6. Import `formatAmount` from `@/lib/connection-labels` for amount formatting in questions.
