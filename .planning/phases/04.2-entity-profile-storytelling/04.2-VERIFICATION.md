## Design Decision Context

After the planned implementation in commit `e696b85`, a deliberate design change was applied in commit `d6a3e36` (titled "refactor: remove inline network graph, show data tables always visible"):

- The inline `NetworkGraph` in story mode was removed (deemed too complex for casual users)
- The `showDetailedRecords` toggle was removed — data tables (`ProfileTabs`) are now **always visible** below story content
- The `onViewDetails` prop on `PatternCallouts` was removed (no longer needed without the toggle)
- The `View details` link inside each callout card was also removed in this same commit

The first two changes are confirmed as intentional per user context. The third change (removal of "View details" links from callout cards) is a **gap** — it removes a required capability (STORY-06: links to source records) that was unrelated to the toggle.

## Gaps Summary

**One gap blocking STORY-06:**

The "View details" link inside PatternCallout cards was removed in commit `d6a3e36` alongside the toggle removal. While the toggle removal was a deliberate UX decision, the link removal appears to be a side-effect cleanup (`"Remove onViewDetails prop from PatternCallouts (no toggle to trigger)"`). However, the link's purpose was to navigate to a connected entity profile — it did not depend on the toggle. The `sourceEntityId` field is still returned by `getPatternCallouts` and is available to the component; it just isn't rendered.

**Fix:** Add a `<Link to="/entity/$id" params={{ id: callout.sourceEntityId }}>View details</Link>` element inside each callout card in `PatternCallouts.tsx`. Requires adding `Link` import from `@tanstack/react-router`.

The other Plan 03 must-have that changed (`showDetailedRecords` toggle, inline NetworkGraph) are documented as intentional design changes, not gaps. The network graph remains accessible in the Visualizations tab inside ProfileTabs.
