    1. Add to `apps/web/src/server-fns/datasets.ts` two new server functions:

       **`getVotingRecord`** — For politician entity profiles (PARL-02). Accepts `{ entityId: string, page?: number }`. Query: join parliamentVoteBallots (where entityId matches) with parliamentVotes (on voteId) and optionally parliamentBills (on billId). Return paginated results ordered by voteDate DESC. Each result: `{ voteDate, subject, billNumber (nullable), billId (nullable), ballotValue, resultName, shortTitleEn (nullable), divisionNumber, parlSessionCode }`. Page size 25.

       **`getBillVotes`** — For bill detail pages (PARL-03). Accepts `{ billId: string }`. Query: join parliamentVotes (where billId matches) with parliamentVoteBallots (on voteId). Return all ballots for the bill grouped by division. For each division: `{ divisionNumber, voteDate, subject, resultName, yeasTotal, naysTotal, ballots: [{ firstName, lastName, caucusShortName, ballotValue, entityId }] }`. Also return the bill record itself (shortTitleEn, longTitleEn, billTypeEn, sponsorEn, currentStatusEn, parlSessionCode) and the billSummary if one exists. No pagination needed — each bill has at most ~10 divisions and ~338 ballots per division.

       **`getBillSummary`** — Accepts `{ billId: string }`. Query billSummaries where billId matches. Return `{ summaryText, model, generatedAt } | null`.

    2. Create `apps/web/src/components/tables/VotesTable.tsx` — following the AidTable/GrantsTable pattern:

       - Use getVotingRecord server function for data
       - Color coding: Yea = green badge, Nay = red badge, Paired = gray badge
       - Bill numbers are links to `/bill/{billId}` when billId is non-null
