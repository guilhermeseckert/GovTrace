# Phase 4.1: How It Works - Context

**Gathered:** 2026-04-03 (assumptions mode)
**Status:** Ready for planning

<domain>
## Phase Boundary

Static explainer page accessible from header and footer navigation. Explains what GovTrace does, where data comes from, how to use it, what AI does, and answers common questions. No data fetching — pure static content. Target: user understands GovTrace in under 60 seconds.

</domain>

<decisions>
## Implementation Decisions

### Route and Navigation
- **D-01:** New route at `/how-it-works` via `apps/web/src/routes/how-it-works.tsx`
- **D-02:** Nav link added to both SiteHeader (alongside Search) and SiteFooter in `__root.tsx`
- **D-03:** Header link text: "How it works" — footer link same

### Page Structure
- **D-04:** Single static component — no loader, no server functions, no data fetching
- **D-05:** Stacked full-width sections using existing container pattern: `mx-auto max-w-7xl px-4 py-16`
- **D-06:** Sections in order: Hero/intro → Data Sources → Step-by-step Walkthrough → AI Transparency → FAQ
- **D-07:** Each section visually distinct with alternating backgrounds (bg-background / bg-muted/30) for scanability

### Data Sources Section
- **D-08:** Show all 5 data sources with name, description, and direct link to the government portal
- **D-09:** Use Lucide icons (not government logos) to avoid licensing issues — e.g., Vote for Elections Canada, FileText for contracts, Gift for grants, Users for lobbying
- **D-10:** Each source card shows: icon, source name, what data it contains, how often it updates, link to official portal

### Step-by-Step Walkthrough
- **D-11:** 3-step visual flow: (1) Search a name → (2) See their connections → (3) Understand the money trail
- **D-12:** Each step shows a brief description of what happens, not screenshots (avoids maintenance burden of keeping screenshots current)
- **D-13:** Use numbered steps with large step indicators and connecting lines/arrows between them

### AI Transparency Section
- **D-14:** Reuse and expand content from `AISummaryExplanation.tsx` — keep messaging consistent
- **D-15:** Explain the three confidence tiers (High ≥85%, Medium 60-85%, Low <60%) matching `ConfidenceBadge.tsx` logic
- **D-16:** Explicitly state: what AI does (entity matching, summary generation) and what it doesn't do (no accusations, no editorial judgment)
- **D-17:** Show visual example of confidence badge with explanation

### FAQ Section
- **D-18:** Accordion/collapsible FAQ items — not all open at once
- **D-19:** Minimum questions: "Is this legal?", "Can I trust the AI?", "What does a connection mean?", "Who built this?", "Where does the data come from?"
- **D-20:** Answers written at grade-school reading level — no jargon

### Copy and i18n
- **D-21:** Inline string literals in JSX for all long-form prose — NOT in en.ts (matches existing pattern for paragraph content in AISummaryExplanation and footer)
- **D-22:** Tone: friendly, plain language, approachable — imagine explaining to your grandpa

### Claude's Discretion
- Exact section spacing and typography sizing
- Whether to add subtle illustrations or keep text-only
- FAQ answer length and phrasing details
- Whether to add a CTA at the bottom ("Try searching now →")

</decisions>
