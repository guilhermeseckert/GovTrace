The lobbyist registry CSV comes from lobbycanada.gc.ca via open.canada.ca. Source URL is the bulk CSV export.

**Source URL:** `https://lobbycanada.gc.ca/app/secure/ocl/lrs/do/clntSmmry?lang=eng&export=true`

Note: If this URL is unavailable, use the open.canada.ca mirror dataset. The downloader should handle a 302 redirect gracefully (follow redirects via `fetch` which does so by default).

**packages/ingestion/src/downloaders/lobby-registrations.ts**:
Same pattern as contracts downloader. Direct CSV download. Returns `{ localPath, fileHash, fileSizeBytes }`.

**packages/ingestion/src/parsers/lobby-registrations.ts**:
Create `LobbyRegistrationRecord` type and `parseLobbyRegistrationsFile` function.

Known column names for lobbyist registrations CSV:
- `registration_number` → registrationNumber (IMPORTANT: use this as `id` directly — stable government key)
- `registrant_name` or `lobbyist_name` or `firm_name` → lobbyistName
- `registrant_type` or `lobbyist_type` or `type` → lobbyistType ('consultant', 'in-house', 'organization')
- `client_name` or `organization_name` → clientName (for consultant lobbyists)
- `subject_matter` or `subject` or `issue` → subjectMatter
- `registration_date` or `start_date` → registrationDate
- `last_updated_date` or `updated_date` or `amend_date` → lastUpdatedDate
- `status` or `reg_status` → status
- `province` or `prov_code` → province

Critical for Pitfall 7: The client_name field must be captured even if it appears under different column names. Log a warning if neither variant is found (the column may not exist for in-house registrations).

ID rule: use `registrationNumber` as the record ID directly (no hash needed — it's a stable government key).

**packages/ingestion/src/upsert/lobby-registrations.ts**:
Same pattern as contracts upsert. BATCH_SIZE = 500. `onConflictDoUpdate` targeting `lobbyRegistrations.id`. Include all non-id fields in `set` block. The `normalizedLobbyistName` and `normalizedClientName` fields are null at ingestion time — they will be populated by the normalizer in Plan 06.
