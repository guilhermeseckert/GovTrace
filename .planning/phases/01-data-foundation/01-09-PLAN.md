<objective>
Fix the broken `onConflictDoUpdate` conflict target in `createNewEntity` in `deterministic.ts`.

Purpose: The current code uses `target: entities.normalizedName` but there is no unique constraint on `normalized_name` in the `entities` table. The only unique index is the composite `entities_canonical_name_type_idx` on `(canonical_name, entity_type)`. PostgreSQL throws "there is no unique or exclusion constraint matching the ON CONFLICT specification" at runtime, crashing every entity creation attempt and breaking the entire matching pipeline (MATCH-01 through MATCH-05).

Output: A one-line fix in `deterministic.ts` that changes the conflict target from `entities.normalizedName` to `[entities.canonicalName, entities.entityType]`. No schema changes needed — the correct unique index already exists.
</objective>
