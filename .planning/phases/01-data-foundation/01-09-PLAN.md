    ```typescript
    .onConflictDoUpdate({
      target: entities.normalizedName,
      set: { canonicalName: rawName },
    })
    ```

    To this:
    ```typescript
    .onConflictDoUpdate({
      target: [entities.canonicalName, entities.entityType],
      set: { normalizedName: normalizeName(rawName) ?? normalizedName, updatedAt: new Date() },
    })
    ```

    Rationale:
    - `entities.normalizedName` has no unique constraint — PostgreSQL rejects it as a conflict target
    - `[entities.canonicalName, entities.entityType]` matches `entities_canonical_name_type_idx` (the only unique index)
    - The `set` clause is updated to keep `normalizedName` and `updatedAt` current on re-insert, which is more useful than setting `canonicalName` (which cannot change — it IS the conflict key)
    - `normalizeName(rawName) ?? normalizedName` safely falls back: `normalizeName` can return empty string but `normalizedName` is already computed at the top of the function; use the already-computed value: `set: { normalizedName, updatedAt: new Date() }`

    Final correct form:
    ```typescript
    .onConflictDoUpdate({
      target: [entities.canonicalName, entities.entityType],
      set: { normalizedName, updatedAt: new Date() },
    })
    ```

    Do not change any other part of the file. The `findDeterministicMatch` function is correct. The `inferEntityType` function is correct.
